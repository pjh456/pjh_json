# 数字与字面量解析

本文说明数字解析的快慢路径，以及 `true/false/null` 的高效匹配方式。

## 数字解析策略
解析器区分两类情况：
1. 纯整数且位数合理
2. 含小数/指数或超长整数

### 快路径：整数
流程：
- 处理可选负号
- 循环累积数字到 `uint64_t`
- 记录位数以避免溢出
- 直接返回 `int64_t`

优势：
- 避免 `std::from_chars` 初始化成本
- 分支少、吞吐高

### 慢路径：浮点或超长
触发条件：
- 出现 `.` 或 `e` 或 `E`
- 或位数超过 18

处理方式：
- 先扫描到数字末尾
- 使用 `std::from_chars` 解析 `double`

保证所有 JSON 数字形式都能正确解析。

## 字面量解析
`true`、`false`、`null` 采用 4/8 字节对比：
- `std::bit_cast` 构造端序相关常量
- `std::memcpy` 读取 4/8 字节避免别名问题
- `false` 使用掩码忽略 padding

这样减少逐字符比较与分支。

未匹配到任何字面量时抛出解析错误。

## 结果类型
解析结果类型映射为：
- `true` / `false` -> `bool`
- `null` -> `std::nullptr_t`
- 整数 -> `int64_t`
- 浮点 -> `double`

这种设计兼顾类型准确性与性能。

## 设计权衡
这套策略重点优化：
- 常见整数解析
- 字面量识别速度

同时确保：
- 浮点与指数形式正确
- 超长整数不会溢出
