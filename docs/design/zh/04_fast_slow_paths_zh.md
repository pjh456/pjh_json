# 快路径与慢路径

本文梳理解析器在各阶段的快慢路径，并解释为何控制流会偏向快路径。

## 总体原则
解析器假设“常见输入”是：
- 以 ASCII 为主
- 转义较少
- 数字长度合理
- 空白格式规整

快路径追求最少分支与 SIMD 友好，慢路径保证特殊情况正确性。

## 空白跳过
快路径：
- `*m_curr > 0x20` 立即返回

慢路径：
- SIMD 批量跳过空白
- 只在发现空白时进入

## 字符串解析
快路径：
- SIMD 扫描快速找到 `"`
- 没有转义时直接返回 `string_view`

慢路径：
- 遇到 `\` 时进入
- 原地重写转义字符
- 处理 Unicode 转义与代理对

错误路径：
- 发现 `< 0x20` 抛错
- 遇到 padding 前未闭合引号抛错

## 数字解析
快路径：
- 纯整数循环累积
- 直接返回 `int64_t`

慢路径：
- 出现 `.`、`e`、`E`
- 或数字过长防止溢出
- 退化为 `std::from_chars` 解析 `double`

## 字面量解析
快路径：
- 4/8 字节直接比较
- `true`/`false`/`null` 一次命中

慢路径：
- 未匹配直接抛错

## 对象与数组
快路径：
- 小容量 `reserve(4)`
- 先占位再覆盖写

慢路径：
- 深度嵌套或大量元素导致多次扩容

## 设计意图
快路径并非“可选优化”，而是 **主要控制流**。
慢路径只用于快路径无法安全处理的少量情况。

这样的结构让解析成本接近理论最小值，同时保证完整正确性。
