# 原地构造策略

本文说明解析器如何原地构造 DOM 节点、减少临时对象，并利用覆盖写实现高效解析。

## 目标
原地构造的设计目标是：
- 避免不必要的临时 `Json`
- 降低拷贝与移动开销
- 减少分配与容量扩展次数

核心策略是：**先占位，再覆盖**。

## 数组解析的占位覆盖
数组解析步骤：
1. 在数组中 emplace 一个 `Json(nullptr)` 作为占位
2. 把该槽位引用传入 `parse_value_inplace`
3. 解析结果直接覆盖占位节点

优势：
- 避免构造临时 `Json` 再 move 进去
- 分支更少，写入路径更短

对应实现：
- `Parser::parse_array_inplace`
- `Parser::parse_value_inplace`

## 对象解析的占位覆盖
对象解析步骤：
1. `emplace_back(key, Json(nullptr))` 预留 value
2. `parse_value_inplace` 直接写入 `entry.second`

收益：
- 每个 value 只构造一次
- DOM 结构早期稳定
- vector 的扩容路径更可控

## 预留容量
数组/对象都会调用 `reserve(4)`：
- 小成本、对小对象收益明显
- 典型 JSON 结构更少扩容

## 字符串的原地重写
若字符串包含转义：
- 进入慢路径，使用 `dst` 写指针原地重写
- 输出字符直接覆盖输入 buffer

结果：
- 不额外分配字符串
- 解析后的字符串就是最终内容

## 设计意义
原地构造可以显著降低：
- 热路径分支
- 临时对象数量
- allocator 压力

这使得用户可以以普通 DOM 方式使用库，
而解析器内部保持高吞吐性能。
