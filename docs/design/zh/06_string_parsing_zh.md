# 字符串解析与 Unicode 处理

本文说明字符串解析的快慢路径、转义处理以及 Unicode 代理对的实现细节。

## 总体流程
字符串解析分两阶段：
1. SIMD 快速扫描特殊字符
2. 遇到转义进入原地重写慢路径

这样既保证常见字符串的高吞吐，又保证语义正确。

## SIMD 快速扫描
解析器使用 `uint8_t` 视角，避免 signed char 的负值干扰。
扫描目标是：
- 终止引号 `"`
- 转义符 `\`
- 非法控制字符 `< 0x20`

匹配结果：
- 首个匹配为 `"`：直接返回 `string_view`
- 首个匹配为 `\`：进入慢路径
- 首个匹配为 `< 0x20`：抛出解析错误

## 原地重写（慢路径）
遇到转义时：
- `dst` 为写指针，`m_curr` 为读指针
- 在同一 buffer 内完成转义替换
- 输出字符串最终连续存储，无需新分配

## 支持的转义
- `\"` `\\` `\/`
- `\b` `\f` `\n` `\r` `\t`
- `\uXXXX`（Unicode）

非法转义会抛出解析错误。

## Unicode 与代理对
处理 `\uXXXX`：
- `parse_hex4()` 解析 4 个十六进制字符

若为高代理（`0xD800` - `0xDBFF`）：
- 期待下一个 `\uXXXX`
- 需要低代理（`0xDC00` - `0xDFFF`）
- 组合为一个完整 codepoint

最后通过 `encode_utf8` 写入 UTF-8 字节序列。

## 控制字符检查
JSON 不允许未转义控制字符：
- SIMD 快速扫描就能发现 `< 0x20`
- 慢路径逐字节再次检查

出现控制字符即抛错。

## 最终字符串视图
解析完成后：
- `std::string_view` 指向原 buffer
- 长度由 `dst` 计算得出

这确保了零拷贝字符串，同时完整支持转义与 Unicode。
